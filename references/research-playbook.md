# Research Playbook

Use this file when the game's lore, proper nouns, tone, setting, or feature vocabulary is not obvious from extracted text alone.

## Source order

Use the highest-signal sources first:
1. official site, publisher site, developer site
2. store pages and official trailers
3. official patch notes or guides
4. community wiki or fandom pages
5. reviews, forum posts, or discussion threads only as supporting evidence

Do not let a fan translation become the canonical truth without corroboration.

## Collect these facts

- genre and core gameplay loop
- setting and tone
- player role
- named factions, towns, regions, facilities
- named characters and relationships
- systems vocabulary: farming, combat, crafting, UI verbs, progression
- item families and material naming patterns
- official capitalization or spacing of proper nouns

## Build the lore packet

Keep the output short and structured:
- `Game summary`
- `Tone and register`
- `Proper nouns`
- `Systems vocabulary`
- `Naming risks`
- `Open questions`

Each fact should be marked as one of:
- `confirmed`
- `probable`
- `uncertain`

If evidence conflicts, keep the term conservative and note the conflict instead of inventing a confident answer.

## Research rules

- Browse before main translation when a string contains a proper noun or implied world context.
- Prefer official spellings for names, places, institutions, and systems.
- If Korean-speaking players already use an overwhelmingly common term, note it as a candidate, not an automatic winner.
- Separate evidence from preference. The packet should explain why a term is chosen.
- Existing human translations are benchmark inputs, not mandatory canonical sources.

## Minimum acceptable output

Even for a small game, the `lore_packet` should answer:
- what world the player is in
- what the player does most often
- which names must stay stable across UI, quests, items, and dialogue
